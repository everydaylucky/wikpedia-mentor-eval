# Wikipedia Mentor Research — Data Collection Pipeline

Academic research pipeline for collecting and analyzing Wikipedia's [Growth Team Mentorship Program](https://www.mediawiki.org/wiki/Growth/Mentor_dashboard) data.

---

## Overview

The program pairs newly registered Wikipedia editors ("mentees") with experienced editors ("mentors"). This pipeline collects the full historical roster of mentors and their conversations with mentees via Wikipedia's public API.

**Pipeline stages:**

```
s1-mentor-collection.py          →  s1_mentor_list.jsonl
                                     s1_mentor_change.jsonl

s2-mentor-mentee-conversation-collection.py  →  s2_mentor_conversation_matched.jsonl
                                                  s2_mentor_conversation_unmatched.jsonl

s3-data-describe.py              →  descriptive statistics (stdout)
```

---

## Scripts

### `s1-mentor-collection.py`
Collects the complete mentor roster (2021-05 to present) from two API sources:
- `Wikipedia:Growth_Team_features/Mentor_list` — wikitext revisions (2021-05 to 2022-10)
- `MediaWiki:GrowthMentors.json` — JSON revisions (2022-10 to present)

Builds a full timeline per mentor including pool status changes, weight history, join/leave events. Resume-safe via checkpoint files.

**Output:**
| File | Description |
|------|-------------|
| `s1_mentor_list.jsonl` | One record per mentor with full timeline |
| `s1_mentor_change.jsonl` | Every state-change event across all mentors |

**Checkpoint files** (gitignored):
- `s1_checkpoint_revisions_wikitext.jsonl`
- `s1_checkpoint_revisions_json.jsonl`
- `s1_checkpoint_uid_map.json`

---

### `s2-mentor-mentee-conversation-collection.py`
Fetches mentor–mentee conversations from Wikipedia User talk pages and archives. For each known mentor, collects their main talk page plus any `/Archive` subpages. Also runs a search API pass to capture talk pages with `"Question from"` headers not belonging to known mentors.

Resume-safe: tracks fetched pages in a checkpoint file. On first run, bootstraps the checkpoint from any existing output files.

**Output:**
| File | Description |
|------|-------------|
| `s2_mentor_conversation_matched.jsonl` | Pages belonging to known mentors |
| `s2_mentor_conversation_unmatched.jsonl` | Pages from search results not matching known mentors |

Each record contains: `mentor`, `page`, `len` (wikitext bytes), `q_count` (question headers found), `wikitext` (raw).

**Checkpoint files** (gitignored):
- `s2_checkpoint_fetched_pages.txt`
- `s2_checkpoint_page_list.json`
- `s2_checkpoint_search_results.jsonl`

---

### `s3-data-describe.py`
Prints descriptive statistics for all collected data. No external dependencies beyond the output files above.

```
python3 s3-data-describe.py
```

## Setup

```bash
pip install python-dotenv
```

Requires a Wikipedia bot password set in a `.env` file (for `s2` only — `s1` uses unauthenticated API):

```
WIKI_BOT_PASSWORD=your_bot_password_here
```

Run in order:

```bash
python3 s1-mentor-collection.py
python3 s2-mentor-mentee-conversation-collection.py
python3 s3-data-describe.py
```

Both `s1` and `s2` are resume-safe — restart at any point and they will skip completed work.

---

## Notes

- Large output files (`s2_mentor_conversation_matched.jsonl` at ~174 MB, `s1_checkpoint_revisions_json.jsonl` at ~102 MB) are excluded from version control via `.gitignore`.
- The "unmatched" category captures mentors who appear in talk page search results but are not in the official mentor registry — likely former mentors or edge cases.
- `q_count` counts `== Question from [[User:...]] ==` section headers as a proxy for individual mentee questions.
